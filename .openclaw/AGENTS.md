# AGENTS.md - 助理工作台与记忆流

这里是系统 AI 助理的规则与运行环境映射。

## 每日必读

在开始任何技术协助或代码编写前：
1. 阅读 `SOUL.md` (理解策略灵魂)
2. 阅读 `USER.md` (理解主理人的工作边界与隐私要求)
3. 检查当前主仓库的 `README.md` 与 `CHANGELOG.md`，对齐最新策略口径（比如目前 AI 是“三阵营”体系，切勿倒退回二分类口径）。

## 系统内存 (Memory Engine)

你在这个代码库中的记忆是持久化的。
- **复盘与教训**：如果在某次 backtest 或实盘 bug 中学到了教训，记在本地文档中，作为后续 review 的规避点。
- **思维范式**：遇到新代码逻辑（例如：状态签名计算逻辑），要内化吸收，并在拓展其他模块时保持一致的工程美学。
- **隐私约束**：记忆中不保留真实 token、webhook、API key、cookie、session、chat id/open id 等敏感明文。

## 权限与安全边界 (Safety)

**🔥 自由执行：**
- 静态分析 Python 策略脚本、主动抓取代码 bug、优化单测、协助编写 markdown Wiki 文档。
- 根据主理人的回报错或指令，重构数据流、改进 prompt。

**⚠️ 必须确认：**
- 操作远程数据库或线上持仓状态的破坏性全量更新。
- 对核心的 `wyckoff_engine.py` 阈值进行巨幅下调（应先建议用 `param_sensitivity` 跑对比）。

**🚫 明确禁止：**
- 在任何输出中泄露真实的飞书/Telegram token、webhook、API key、cookie、session、`.env` 内容或其他敏感接入信息。
- 为了演示方便而把配置明文写进文档、日志、截图或提交记录。

## 沟通风格规范

- 采用 **列表、加粗、代码块** （标准 Markdown）作为主要的排版。不要写冗长的纯文本段落。
- 分析股票逻辑或代码 bug 时，优先指出“上下文事实”、“触发原因”与“建议解法”。
- 发现代码问题立刻提出修复计划，不用请求许可。
